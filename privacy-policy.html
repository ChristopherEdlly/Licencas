Política de Privacidade

Dashboard de Licenças Prêmio - SEFAZ-SE

Última atualização: 12 de novembro de 2025

Introdução

Esta Política de Privacidade descreve como o Dashboard de Licenças Prêmio da Secretaria de Estado da Fazenda de Sergipe (SEFAZ-SE) coleta, usa, armazena e protege suas informações pessoais.

Ao utilizar este sistema, você concorda com as práticas descritas nesta política.

Dados Coletados

Dados de Autenticação

  • Email corporativo: Seu endereço @fazenda.se.gov.br para autenticação via Microsoft Identity Platform
  • Nome de usuário: Nome associado à sua conta Microsoft corporativa
  • ID de usuário: Identificador único fornecido pela Microsoft

Dados da Planilha

  • Informações de licenças: Dados sobre cronogramas de licenças-prêmio dos servidores
  • Dados de servidores: Matrícula, nome, cargo, lotação, datas de licença

Dados Técnicos

  • Cache local: Arquivos carregados são armazenados temporariamente no seu navegador para melhorar a experiência
  • Preferências: Configurações de tema, filtros e visualização salvos localmente
Importante: Todos os dados permanecem no seu navegador (localStorage). Não enviamos dados para servidores externos, exceto a conexão com SharePoint da Microsoft.

Como Usamos Seus Dados

Os dados coletados são utilizados exclusivamente para:

  • Autenticação: Verificar sua identidade e garantir acesso apenas a funcionários da SEFAZ-SE
  • Visualização: Exibir cronogramas e estatísticas de licenças-prêmio
  • Análise: Gerar relatórios e análises sobre o planejamento de licenças
  • Funcionalidade: Salvar preferências de filtros, tema e configurações
NÃO usamos seus dados para:
  • Marketing ou publicidade
  • Compartilhamento com terceiros
  • Venda de informações
  • Análise de comportamento para fins comerciais

Segurança e Proteção

Medidas de Segurança

  • Acesso restrito: Apenas emails @fazenda.se.gov.br podem acessar o sistema
  • Autenticação Microsoft: Utiliza a plataforma segura Microsoft Identity (Azure AD)
  • HTTPS: Todas as conexões são criptografadas via SSL/TLS
  • Armazenamento local: Dados ficam apenas no seu navegador, não em servidores
  • Tokens temporários: Tokens de autenticação expiram automaticamente

SharePoint Integration

A conexão com SharePoint da Microsoft segue as melhores práticas:

  • Permissões mínimas necessárias (apenas leitura de arquivos)
  • OAuth 2.0 para autenticação segura
  • Tokens de acesso de curta duração

Compartilhamento de Dados

Transparência Total:

Não armazenamos dados em nossos servidores, portanto, não temos dados para compartilhar. Todo processamento acontece no seu navegador (client-side).

Serviços de Terceiros Utilizados

Os únicos serviços externos que podem ter acesso a dados são:

  • Microsoft (Azure AD): Apenas para autenticação (email e nome)
  • Microsoft SharePoint: Onde a planilha corporativa está armazenada (pela SEFAZ-SE)
  • GitHub Pages: Hospeda os arquivos estáticos do site (HTML, CSS, JS) - não tem acesso a dados

Fluxo de Dados

Para sua compreensão completa:

  1. Você faz login via Microsoft (Azure AD da SEFAZ-SE)
  2. O navegador recebe um token temporário de acesso
  3. Com esse token, o navegador busca a planilha do SharePoint
  4. Os dados são processados localmente no seu navegador
  5. Nenhum dado passa por servidores da SEFAZ-SE

Políticas de Privacidade de Terceiros

  • Microsoft: Declaração de Privacidade da Microsoft
  • GitHub: Política de Privacidade do GitHub

Seus Direitos e Controle de Dados

Importante: Como todos os dados ficam armazenados apenas no seu navegador (client-side), você tem controle total sobre eles. Não armazenamos nenhuma informação em servidores.

Como Gerenciar Seus Dados

  • Limpar cache local: Vá em Configurações do navegador → Limpar dados de navegação → Dados em cache
  • Remover preferências: Use as configurações do sistema ou limpe o localStorage do navegador
  • Revogar autenticação: Faça logout da sua conta Microsoft corporativa
  • Excluir tudo: Basta limpar os dados do site nas configurações do navegador

Direitos LGPD

Como não armazenamos dados em servidores, os seguintes direitos podem ser exercidos diretamente por você:

  • Acesso: Inspecione o localStorage do navegador (F12 → Application → Local Storage)
  • Exclusão: Limpe os dados do site nas configurações do navegador
  • Portabilidade: Exporte seus relatórios através das funcionalidades do sistema
  • Revogação: Logout e limpeza de cache removem sua autenticação

Dados na Microsoft

Os únicos dados que não controlamos diretamente são os gerenciados pela Microsoft:

  • Autenticação: Gerenciada pelo Azure AD da SEFAZ-SE
  • SharePoint: Planilha armazenada no SharePoint corporativo

Para questões sobre esses dados, entre em contato com o TI da SEFAZ-SE ou acesse: Privacidade Microsoft

Retenção de Dados

  • Cache local: Mantido até que você limpe o cache ou desinstale o navegador
  • Preferências: Armazenadas indefinidamente no localStorage do navegador
  • Tokens de autenticação: Expiram automaticamente após período de inatividade

Você pode limpar todos os dados locais a qualquer momento através das configurações do navegador.

Cookies e Armazenamento Local

Utilizamos armazenamento local (localStorage) do navegador para:

  • Salvar preferências de tema (claro/escuro)
  • Manter filtros e configurações ativas
  • Armazenar cache de arquivos carregados
  • Guardar tokens de autenticação Microsoft

Não utilizamos cookies de rastreamento ou publicidade.
